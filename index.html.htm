SELECT
  CASE WHEN SHP_STEP_ORIGIN_ID IN (
    'BRXSP9','ARENA','BRXSP10','BRSP14','XMG1','CAMPINAS',
    'BRSP02','BRSP04','BRXGO1','XSP4','BRXSP5','BRBA01','BRBA02'
  ) THEN SHP_STEP_ORIGIN_ID ELSE 'Outras' END AS facility,
  DATE_CPT AS data,
  CASE
    WHEN SHP_SERVICE_ID IN (3966010,3022162,966283) THEN 'J&T'
    WHEN SHP_SERVICE_ID = 135551 THEN 'Loggi'
    WHEN SHP_SERVICE_ID IN (263,264) THEN 'Jad'
    WHEN SHP_SERVICE_ID = 251165 THEN 'Carvalima'
    WHEN SHP_SERVICE_ID = 4692270 THEN 'iMile Slow'
    WHEN SHP_SERVICE_ID = 4675820 THEN 'iMile Fast'
    WHEN SHP_SERVICE_ID = 4732666 THEN 'Dominalog'
    WHEN SHP_SERVICE_ID = 157861 THEN 'iMile'
    ELSE 'Outros'
  END AS carrier,
  COUNT(DISTINCT SHP_SHIPMENT_ID) AS volume
FROM `meli-bi-data.SBOX_MLBCOMCARRIERS.Artefato_previsao_coleta`
WHERE DATE_CPT BETWEEN
  DATE_SUB(CURRENT_DATE(), INTERVAL 28 DAY)
  AND DATE_ADD(CURRENT_DATE(), INTERVAL 10 DAY)
GROUP BY 1, 2, 3
ORDER BY data, facility, carrier

Cálculo MOD: MOD (Média) = Média Diária ÷ Produtividade/Turno · MOD (Pico) = Dia Pico ÷ Produtividade/Turno.
Turnos (Pico): Dia Pico ÷ Pcts/Turno, arredondado para cima. O campo de placeholder já sugere, a título de exemplo, o Pcts/Turno necessário para fechar em 3 turnos. Até 3 = turnos normais (verde) · 4 = 3 turnos + hora extra (amarelo) · acima de 4 = ⚠ fora da capacidade com a config atual de 3 turnos + 1 HE (vermelho).
Desvio de Volume (%): campo nos Filtros Globais (sidebar) para simular uma folga/acréscimo percentual sobre o volume. Preenchido: o valor ajustado passa a ser usado nos cálculos de Média, Pico, MOD e Turnos, e aparece em azul entre parênteses embaixo de cada número (ex: "(+10%: 84.599)"). Vazio ou zerado: tudo volta a considerar o volume original de base. As cores do heatmap (farol) sempre refletem o volume original, independente do desvio configurado.
Clique no cabeçalho de um dia (Seg...Sáb) para ordenar as origens por aquele dia. "Outras" agrega origens fora do acompanhamento individual, para o total geral bater com a fonte. Este arquivo pode rodar em modo "planilha viva": configure SHEET_CSV_URL no código com a URL publicada em CSV de uma planilha conectada ao BigQuery, e o dashboard passa a buscar os dados atualizados automaticamente a cada abertura.
